# `thekernel-ax` 0.1.0 release notes

This checkpoint prepares the first independent releases of
`thekernel-axsched`, `thekernel-axpoll`, and `thekernel-axtask`.

## `thekernel-axsched` 0.1.0

- Establishes a new package identity while retaining the `axsched` Rust library
  name.
- Carries the maintained FIFO, round-robin, fair, and real-time scheduling
  mechanisms extracted from TheKernel.
- Keeps process lifecycle policy out of scheduler state: Linux-style
  reset-on-fork is a downstream ABI/task-layer responsibility.
- Uses the full nonzero `u8` real-time priority domain and leaves narrower ABI
  ranges to adapters.
- Makes removal of a task owned by another FIFO/RR scheduler safe and
  side-effect free.
- Makes priority updates, CFS task configuration, and fair-child vruntime
  seeding distinguish unsupported operations, invalid values, incompatible
  classes, busy transactions, and foreign queue ownership.
- Publishes each CFS class/nice/real-time-priority tuple atomically so
  concurrent readers never see a torn class-specific state.
- Builds from registry dependencies without TheKernel's patch table.

## `thekernel-axpoll` 0.1.0

- Establishes a new package identity while retaining the `axpoll` Rust library
  name.
- Removes the core dependency on `linux-raw-sys`; Linux `POLL*` translation now
  belongs to the caller's ABI adapter.
- Replaces inherited event abbreviations with neutral `IoEvents` names and
  crate-owned bit values.
- Adds const-generic bounded `PollSet` capacity, explicit `Full` and `Closed`
  errors, opaque generation tokens, cancellation, waker update, and terminal
  closure.
- Adds a per-source prepare/arm seam that returns unpublished waker ownership
  on failure, so consumers can reserve, arm, and roll back a finite topology
  without treating one token as an aggregate.
- Gives every `register` call an independent token even for equivalent wakers;
  one wait cannot accidentally cancel another that shares an executor waker.
- Removes the inherited `Pollable` trait because a single token cannot express
  cancellation of a wait that fans in to multiple readiness sources. This
  release guarantees per-source lifecycle only; downstream object/fd layers
  retain bounded composite registration and rollback policy.
- Prevents ABA cancellation after slot reuse and rejects tokens from another
  registry.
- Guarantees no silent replacement at capacity and keeps RawWaker clone, drop,
  and wake behavior outside the IRQ-safe linearization lock.
- Enables the registry lock's SMP implementation explicitly rather than
  depending on another workspace member to unify the `kspin/smp` feature.
- Removes the empty `alloc` feature; the crate's allocator requirement is
  unconditional and explicit.

### Migration from inherited `axpoll`

Callers must retain the token returned by `register`, use `update` whenever the
same future is polled again, and call `cancel` when the wait is abandoned. They
must also replace the old event names according to the table in the crate
README and perform Linux bit translation outside this crate.

`wake()` still consumes current registrations but no longer implies terminal
state. Call `close()` when no future registrations are valid.

Do not migrate a multi-source `Pollable::register` implementation by returning
only one of its tokens. Its consumer needs an explicit bounded registration
group (or equivalent owner) that cancels every source and rolls back partial
failure.

## `thekernel-axtask` 0.1.0

- Retains generic task, run-queue, wait-queue, timer, IRQ-wake, affinity, and
  scheduler-integration mechanisms while removing Linux PID policy.
- Makes task identity and scheduler publication exhaustion explicit.
- Replaces aliased global mutable runqueue references with guarded per-CPU
  ownership.
- Uses owner-CPU, generation-checked timer tokens so migration cannot cancel a
  different CPU's timer.
- Bounds timer futures, persistent timer callbacks, distinct IRQ sources, and
  IRQ waiters; registration and cancellation have typed outcomes.
- Keeps IRQ-domain validation, enable/disable, masking, and acknowledgement in
  the driver/capability owner; waiter registration has no hardware side effect.
- Runs RawWaker clone, destruction, and callbacks outside IRQ-safe locks.
- Removes object-level `Pollable::poll_io` policy from the task crate; consumers
  use `thekernel-axpoll` plus an ABI/object adapter.
- Replaces unbounded remote handoff spinning with an explicit state machine and
  exactly-one-Arc ownership transfer.
- Makes the public current-task handle own its own Arc reference, closing the
  lifetime hole that allowed safe code to retain a non-owning per-CPU pointer
  across later task reclamation, while releasing internal handles before a
  non-returning exit switch so task objects and kernel stacks remain
  reclaimable.
- Admits affinity only when the mask intersects initialized run queues and does
  not publish runnable tasks onto possible-but-offline CPUs.
- Adds a condition-aware interruptible timed wait backed by one full deadline,
  closing the need for hidden 10 ms signal-polling loops in consumers.
- Adds fallible CFS publication reservation before lifecycle visibility, with
  owner-carrying failure/cancellation and allocation-free final linking. The
  token fixes the runqueue/ordering admission and excludes affinity retargeting
  until commit or cancellation.
- Keeps the idle task in Running state while it probes the scheduler and treats
  that state as a valid later empty-queue fallback without publishing fake
  readiness.
- Replaces the allocating exited-task queue with task-embedded intrusive links,
  bounded reclaim batches, and durable ownership-fault diagnostics.
- Returns typed scheduling-update and raw-state-decode failures instead of
  ambiguous booleans or consumer-triggerable conversion panics.
- Retains the inherited optional `tls` feature name only as a compile-time
  rejection for forwarding-manifest compatibility until the lower `axhal`
  layer offers a fallible TLS-area allocation contract; 0.1.0 does not execute
  a task-creation path that bypasses typed OOM handling.

## Verification

All three packages are gated by formatting, their declared MSRV or pinned
nightly, clippy with warnings denied, rustdoc, RV64/LA64 checks where
applicable, a registry-only dependency audit, provenance checks, and tests
executed from unpacked `.crate` artifacts.
