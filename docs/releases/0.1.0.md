# `thekernel-ax` 0.1.0 release notes

This checkpoint prepares the first independent releases of
`thekernel-axsched` and `thekernel-axpoll`. It does not publish or push them.

## `thekernel-axsched` 0.1.0

- Establishes a new package identity while retaining the `axsched` Rust library
  name.
- Carries the maintained FIFO, round-robin, fair, and real-time scheduling
  mechanisms extracted from TheKernel.
- Builds from registry dependencies without TheKernel's patch table.

## `thekernel-axpoll` 0.1.0

- Establishes a new package identity while retaining the `axpoll` Rust library
  name.
- Removes the core dependency on `linux-raw-sys`; Linux `POLL*` translation now
  belongs to the caller's ABI adapter.
- Replaces inherited event abbreviations with neutral `IoEvents` names and
  crate-owned bit values.
- Adds const-generic bounded `PollSet` capacity, explicit `Full` and `Closed`
  errors, opaque generation tokens, cancellation, waker update, and terminal
  closure.
- Gives every `register` call an independent token even for equivalent wakers;
  one wait cannot accidentally cancel another that shares an executor waker.
- Removes the inherited `Pollable` trait because a single token cannot express
  cancellation of a wait that fans in to multiple readiness sources. This
  release guarantees per-source lifecycle only; downstream object/fd layers
  retain bounded composite registration and rollback policy.
- Prevents ABA cancellation after slot reuse and rejects tokens from another
  registry.
- Guarantees no silent replacement at capacity and keeps RawWaker clone, drop,
  and wake behavior outside the IRQ-safe linearization lock.

### Migration from inherited `axpoll`

Callers must retain the token returned by `register`, use `update` whenever the
same future is polled again, and call `cancel` when the wait is abandoned. They
must also replace the old event names according to the table in the crate
README and perform Linux bit translation outside this crate.

`wake()` still consumes current registrations but no longer implies terminal
state. Call `close()` when no future registrations are valid.

Do not migrate a multi-source `Pollable::register` implementation by returning
only one of its tokens. Its consumer needs an explicit bounded registration
group (or equivalent owner) that cancels every source and rolls back partial
failure.

## Verification

Both packages are gated by formatting, MSRV/stable workspace tests, clippy with
warnings denied, a registry-only dependency audit, and tests executed from the
unpacked `.crate` artifacts.
