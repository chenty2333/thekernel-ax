# Security Policy

## Supported versions

The latest release in the `0.1.x` line receives security fixes. Older
development snapshots may be fixed only when the correction can be applied
without obscuring the current maintained history.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private
security-advisory flow for
<https://github.com/chenty2333/thekernel-ax/security/advisories/new>. Include
the affected crate and version, a minimal reproducer if available, the expected
impact, and whether the issue involves unsafe code, unbounded resource use, a
lost wakeup, stale registration, or a lock/callback re-entrancy path.

Maintainers will acknowledge a complete report as soon as practical, assess
severity and affected versions, coordinate a fix and disclosure date, and
credit the reporter unless anonymity is requested.

Public issues are appropriate for ordinary correctness bugs that do not expose
a security or embargo concern.
